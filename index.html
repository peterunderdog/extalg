The Ext-algebra Page

Introduction

Welcome to the Ext-algebra page! Using links on this page, you may

  • define a special biserial algebra by drawing its quiver and relations using a Java applet ( quiverCAD)
  • use a web application to compute the basis of the Ext-algebra of your algebra using the method developed in The Ext-algebra of a Representation-Finite Biserial Algebra, Journal of Algebra 221 611-629 (1999)
  • View the quiverCAD Java API

For more information, follow one of these links:

  • Using quiverCAD
  • Examples
  • Compute an Ext-algebra

Warning

This software is still in its infancy so it may not be well-behaved at all times. Some symptoms of bad behavior may include

  • incorrect results
  • unexpected results
  • failure to return results (infinite loop)

Requirements

In order to run the quiverCAD applet, you will need the Java 1.3 plug-in, which is available for download from Sun's Java site. If the plug-in is not installed, you will be prompted to download and install it when you try to run the applet.

The applet does not appear to work on Macs at the moment )-:

Notes on Ext-algebra computation

The quiverCAD applet does not yet check that the relations defined on the quiver satisfy the special biserial conditions but the computational engine that calculates the Ext-algebra makes some implicit assumptions about the input it receives. Therefore, if the quiver submitted for computation is not special biserial, some implicit relations will be imposed on it to make it special biserial. In particular, any two parallel paths will be assumed to be equal. In a future version of quiverCAD the special biserial conditions will be checked whenever an arrow is added to the quiver, and whenever relations are deleted to ensure that the quiver always represents a special biserial algebra.

The Ext-algebra computational engine terminates non-zero cycles at length 62 to prevent infinite loops.

Basis elements which are cyclic (i.e., long exact sequence beginning and ending with the same simple module) are not extended further by the program, so the actual basis contains all products of these sequences. The products will not be contained in the set of minimal generators anyway.

Special biserial algebras of infinite representation type are identified by the Ext-algebra engine by looking for primitive V-sequences, that is string modules which can be joined arbitrarily many times with themselves to yield indecomposable modules of arbitrary length. The algorithm for computing Ext-algebras does not apply to the case of infinite representation type.
